# /standup

直近の git アクティビティからスタンドアップ報告を生成する。

## 動作

1. **直近 24h の git log を読む**
   - `git log --since="24 hours ago" --oneline --author="$(git config user.name)"`
   - コミットがなければ作業中のファイル（`git status`）を確認

2. **未完了タスクを確認**
   - `logs/weekly/W{N}.md` の未チェックタスク（`- [ ]`）を抽出

3. **3 セクションで出力**
   - **昨日やったこと**: コミット内容を要約（1-3 行）
   - **今日やること**: 未チェックタスクの上位 3 項目
   - **ブロッカー**: 詰まっていることがあれば明示

## 出力フォーマット

```
## スタンドアップ — YYYY-MM-DD

**昨日**
- {コミット要約 or 作業内容}

**今日**
- {タスク1}
- {タスク2}
- {タスク3}

**ブロッカー**
- なし / {詰まっていること}
```

## 使用例

```
/standup
```

毎朝の作業開始時に実行。チームへの共有文面として使うか、個人の作業ログとして `logs/weekly/W{N}.md` に貼り付ける。

## 運用ルール

- 朝ブロック開始前に実行して今日の優先順位を確認する
- ブロッカーが 2 日以上続く場合は設計を見直すサイン

## 関連

- `logs/weekly/W{N}.md` — 週次ログ（タスクリストの参照元）
- `/weekly-review` — 週末の振り返り
- `/debug` — ブロッカーが技術的問題の場合
