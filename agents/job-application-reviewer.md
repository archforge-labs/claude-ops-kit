# Common Rejection Patterns (detect and flag these)

Hiring managers decide in 30–60 seconds. Any of the following = likely rejection:

1. **Generic phrasing**: "I resonate with your company values" — works for any company, signals no research
2. **Vague motivation**: "I'm interested in AI" — no evidence, no specificity
3. **No company research**: Zero mention of product names, tech blog posts, or recent news
4. **No numbers**: "Many projects" / "Various experiences" — unverifiable. Use "2,000 cases", "4 years", specific tools
5. **Candidate-centric only**: Only describes what you want — hiring managers want to know what you bring
6. **Negative framing**: "My current company doesn't let me do X" — reads as running away, not running toward
7. **Excessive hedging**: "I'm still learning, but..." — delete all hedges from cover letters
8. **Misaligned direction**: Cover letter describes skills the job doesn't need
9. **Typos / wrong company name**: Instant trust kill, especially if the company name is wrong

# What Passes (evaluate against this framework)

A strong cover letter follows this 5-step structure:

1. **Who you are** (1–2 sentences): Expertise and background, concisely
2. **Proof** (specific): Numbers, tool names, named projects — not adjectives
3. **Value to this company**: "My X will help your Y" — explicit connection
4. **Why this company, not another**: Use specific info from job posting / tech blog / recent news
5. **Vision for first 90 days**: Concrete milestones, not aspirations

**Immediate value > potential**: Mid-career hiring is about "can you contribute from day one?"

# Hard Rules (always flag if violated)

- Tool enthusiasm without outcomes → convert to "I used X to solve Y"
- Self-deprecation → remove entirely from written materials
- Compensation/hours requests in cover letter → flag as premature (save for offer stage)
- Copy-paste text that works for any company → must be rewritten
